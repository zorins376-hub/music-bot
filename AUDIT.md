## Фаза 1 — Безопасность и целостность данных

### CRITICAL

#### #1. CORS открыт для всех origins

| | |
|---|---|
| **Файл** | `webapp/api.py:256-260` |
| **Проблема** | `allow_origins=["*"]` позволяет любому сайту делать запросы к API |
| **Решение** | Ограничить origins списком разрешённых доменов (домен TMA, админка) |

#### #2. Захардкоженный Supabase JWT-токен в коде

| | |
|---|---|
| **Файл** | `bot/services/supabase_mirror.py:23-32` |
| **Проблема** | Реальный JWT-токен используется как default fallback если env не задан |
| **Решение** | Убрать default-значение, сделать обязательным через env или отключать mirror если ключа нет |

#### #3. Дефолтные admin-креденшалы "admin"/"admin"

| | |
|---|---|
| **Файл** | `webapp/admin_api.py:66-70` |
| **Проблема** | `os.environ.get("ADMIN_PASSWORD", "admin")` — если env не задан, логин доступен всем |
| **Решение** | Убрать дефолтные значения, требовать наличие env-переменных. Добавить валидацию при старте |

#### #4. Deploy .env файлы в git с реальными токенами

| | |
|---|---|
| **Файлы** | `deploy/.env.vps`, `deploy/.env.production` |
| **Проблема** | `ADMIN_TOKEN=musicbot-admin-secret-2026` — реальный токен в репозитории |
| **Решение** | Переименовать в `.env.vps.example` / `.env.production.example`, убрать реальные значения, добавить в `.gitignore` |

#### #5. Нет rate-limiting на admin логин

| | |
|---|---|
| **Файл** | `webapp/admin_api.py:104-110` |
| **Проблема** | Можно бесконечно брутфорсить пароль админки |
| **Решение** | Добавить rate-limiting (5 попыток за 5 минут), lockout после превышения |

#### #6. Эндпоинт /api/errors без проверки авторизации

| | |
|---|---|
| **Файл** | `webapp/api.py:468-479` |
| **Проблема** | Доступ к логам ошибок без проверки прав |
| **Решение** | Добавить admin-авторизацию на этот эндпоинт |

#### #7. asyncio.gather() без return_exceptions — один провайдер роняет весь поиск

| | |
|---|---|
| **Файлы** | `bot/handlers/inline.py`, `bot/handlers/search.py:410`, `webapp/api.py:1131` |
| **Проблема** | Если один из 5 провайдеров (Yandex, Spotify, VK, YouTube, local) упадёт — пользователь не получит результаты ни от одного |
| **Решение** | Добавить `return_exceptions=True` и фильтровать исключения, возвращая частичные результаты |

#### #8. Отсутствие CASCADE DELETE на Foreign Keys

| | |
|---|---|
| **Файлы** | `bot/models/daily_mix.py:16`, `bot/models/favorite.py:13`, `bot/models/party.py`, `bot/models/recommendation_log.py:19`, `bot/models/playlist.py:29` |
| **Проблема** | При удалении пользователя или трека в БД остаются сиротские записи. 7+ моделей без `ondelete="CASCADE"` |
| **Решение** | Добавить `ondelete="CASCADE"` на все FK → users.id и tracks.id, написать миграцию |

#### #9. Нет системы миграций БД

| | |
|---|---|
| **Файл** | `docs/migrations/001_user_columns.sql` — ручной SQL |
| **Проблема** | Нет Alembic, нет версионирования схемы. Изменения схемы применяются вручную |
| **Решение** | Подключить Alembic, сгенерировать initial migration из текущих моделей |

## Фаза 2 — Стабильность и конкурентность

### HIGH

#### #10. Монолитный webapp/api.py (213KB)

| | |
|---|---|
| **Файл** | `webapp/api.py` |
| **Проблема** | Один файл 213KB с API, кешами, утилитами, бизнес-логикой |
| **Решение** | Разбить на модули: `routes/`, `services/`, `cache.py`, `middleware.py` |

#### #11. Монолитный Player.tsx (2466 строк)

| | |
|---|---|
| **Файл** | `webapp/frontend/src/components/Player.tsx` |
| **Проблема** | 69 пропсов, 50+ state-переменных, вся логика в одном компоненте |
| **Решение** | Вынести: PlayerCore, AudioVisualizer, EQPanel, AudioBadges как отдельные компоненты |

#### #12. SSE reconnect без backoff и лимита

| | |
|---|---|
| **Файл** | `webapp/frontend/src/components/PartyView.tsx:114-173` |
| **Проблема** | Рекурсивный reconnect через 2с без exponential backoff и без max retries |
| **Решение** | Добавить exponential backoff (2s → 4s → 8s...) и max retries (10) |

#### #13. Тихое проглатывание ошибок по всему бэкенду

| | |
|---|---|
| **Файлы** | `bot/db.py:173-230`, `bot/services/supabase_mirror.py:76-92` и множество мест |
| **Проблема** | `try/except pass` — ошибки теряются, баги не обнаруживаются. 1448 `except Exception` + 60 bare `except:` |
| **Решение** | Заменить `pass` на `logger.exception()`, убрать bare `except:`, оставить только осознанные обработки |

#### #14. Race condition с файлом cookies

| | |
|---|---|
| **Файл** | `bot/services/downloader.py:153-161` |
| **Проблема** | Несколько воркеров одновременно копируют/пишут в один файл cookies |
| **Решение** | Использовать копию файла per-worker или файловую блокировку |

#### #15. Нет Error Boundaries на фронтенде

| | |
|---|---|
| **Файлы** | Все компоненты кроме `ProfileView.tsx` |
| **Проблема** | Ошибка в одном компоненте ломает всё приложение |
| **Решение** | Обернуть ключевые маршруты в ErrorBoundary (Player, PartyView, LiveRadio, Playlists) |

#### #16. Fire-and-forget без обработки ошибок

| | |
|---|---|
| **Файлы** | `webapp/frontend/src/api.ts:320-348`, `bot/main.py:96` |
| **Проблема** | `asyncio.create_task()` без done-callback, `ingestEvent()` без await |
| **Решение** | Добавить error-callback на все create_task, логировать ошибки |

#### #17. Глобальный мутабельный стейт без локов

| | |
|---|---|
| **Файл** | `bot/handlers/search.py:171` |
| **Проблема** | `_group_sessions: dict = {}` — несколько корутин могут одновременно читать/писать |
| **Решение** | Обернуть в `asyncio.Lock()` или использовать thread-safe структуру |

#### #18. Мутация приватного атрибута ThreadPoolExecutor

| | |
|---|---|
| **Файл** | `bot/services/download_manager.py:206` |
| **Проблема** | `self._pool._max_workers = new_size` — хак, сломается при обновлении Python |
| **Решение** | Пересоздавать пул с новым размером вместо мутации internal state |

#### #19. Event listeners без cleanup → memory leak

| | |
|---|---|
| **Файл** | `webapp/frontend/src/App.tsx:350-370` |
| **Проблема** | `audio.addEventListener("playing/pause/seeked")` без `removeEventListener` в cleanup useEffect |
| **Решение** | Вернуть cleanup-функцию из useEffect, которая снимает все listener'ы |

#### #20. setInterval без clearInterval

| | |
|---|---|
| **Файл** | `webapp/frontend/src/App.tsx:774` |
| **Проблема** | `loudnessTimerRef` — интервал не всегда очищается при размонтировании |
| **Решение** | Добавить `clearInterval(loudnessTimerRef.current)` в cleanup |

#### #21. Shutdown без таймаута

| | |
|---|---|
| **Файл** | `bot/main.py:190-215` |
| **Проблема** | `on_shutdown()` закрывает сессии без общего таймаута. Если Redis/Supabase завис — бот не остановится |
| **Решение** | Обернуть shutdown в `asyncio.wait_for(..., timeout=10)`, cancel все background tasks |

## Порядок работы и верификация

### Рекомендуемые фазы

| Фаза | Задачи | Фокус | Ожидаемое время |
|-------|--------|-------|-----------------|
| **Фаза 1** | #1 — #9 | Безопасность + целостность данных | 2-3 дня |
| **Фаза 2** | #10 — #21 | Стабильность + конкурентность | 3-5 дней |
| **Фаза 3** | #22 — #38 | Производительность + DevOps | 5-7 дней |
| **Фаза 4** | #39 — #50 | Качество кода + DX | 3-4 дня |

### Верификация после каждой фазы

1. Запустить бота локально, проверить основные сценарии (поиск, скачивание, плейлисты)
2. Проверить webapp через Telegram Mini App (авторизация, плеер, party mode)
3. Прогнать `docker-compose up` и убедиться что всё стартует
4. Запустить тесты: `pytest tests/`
5. Для security-фиксов: попробовать атаку до и после (CORS запрос с чужого домена, брутфорс логина)

### Топ-5 самых опасных проблем

| # | Проблема | Риск |
|---|----------|------|
| #7 | `asyncio.gather` без `return_exceptions` | Падение одного провайдера ломает поиск для всех пользователей |
| #1 | CORS `allow_origins=["*"]` | Любой сайт может выполнять запросы от имени пользователя |
| #3 | Дефолтный пароль admin/admin | Полный доступ к админке без настройки env |
| #8 | Нет CASCADE DELETE | Накопление мусорных данных, нарушение целостности |
| #13 | 1500+ проглоченных исключений | Невидимые баги, невозможность диагностики проблем |
